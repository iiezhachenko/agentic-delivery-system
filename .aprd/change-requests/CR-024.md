# CR-024 — Context projector: hint→field-selector closes L7

> Standalone. Operator-directed (2026-06-13). Workstream: `feature/mcp-adp-powers`. Source: `_self-host/00-analysis.md` §3.3. Caveman register.

## Ask

L7 = highest-weight token leak. Step-runner cold-reads resolved input files whole. Measured (IMPLEMENT mode=skeleton-build): ~17 files — 6 ADR bodies + components/contracts/data-model JSON + oracle.json + 3 test files + build-plan + locks. Runner ingests all of them raw (~40–60k tokens).

`io-manifest.json` ALREADY declares per-input `hint` per input ("R*=responsibilities/trace", "CT* for your seams") — the projection selector exists but is unused. `resolve.mjs` returns paths; runner reads whole files.

Fix: `adp_next` (CR-023) reads files server-side and **projects** each to the fields its hint names before shipping. Model gets slices, not whole files. Default = whole file when no selector (back-compat, safe migration).

## Scope delta

### A — Extend `io-manifest.json` with optional `project:` field

Each input entry gains optional `"project": "<selector-expr>"`. Selector syntax = minimal: dot-path `foo.bar`, key-pattern `CT*`, or array-filter `[class=bugfix]`. Default: absent → ship whole file. Example:
```json
{ "path": ".hld/skeleton/contracts.json", "hint": "CT* for your seams", "project": "CT*" }
```
Back-compat: existing entries without `project:` field behave identically to today.

### B — New module `tools/io/project.mjs`

Pure function: `project(fileContents, selectorExpr) → slicedContent`. Reads JSON / text; applies selector; returns only matching subset. Deterministic — no model call. Covers: key-pattern match on JSON objects, dot-path extraction, array filter. Non-JSON text: selector ignored, whole content returned (safe fallback).

### C — `adp_next` calls projector per input

After resolving input paths (existing `resolve.mjs`), `adp_next` reads each file + calls `project.mjs` with that entry's selector. Ships projected slice to model, not raw file path. `inputs[]` in the step packet carries `{path, slice}` — model reads slice, not disk directly.

### D — Both-directions selftest for projector

`tools/io/project.selftest.mjs`: planted "need a dropped field" case MUST FAIL (confirms under-projection is caught). Whole-file case MUST PASS. Key-pattern + dot-path + array-filter all tested. Same bar as every spine module.

### E — `io-manifest.json` migration (back-compat wave)

Add `project:` selectors to high-value entries first: IMPLEMENT inputs (largest files). Low-value / small files: leave absent (ship whole, no regression). Document selector syntax in `io-manifest.json` schema header comment.

## Affected downstream

- `io-manifest.json`: gains optional `project:` field per entry (schema-additive, back-compat).
- `adp_next` (CR-023): calls `project.mjs` per resolved input.
- All step-runner invocations: receive smaller packets → context drops from ~40–60k input tokens → ~3–5k per IMPLEMENT step.
- `graph-lint.selftest.mjs` / `resolve.selftest.mjs`: unchanged (resolve still returns paths; projection is server-side, post-resolve).

spine-extension. New module `project.mjs` + io-manifest schema extension + `adp_next` wiring.
