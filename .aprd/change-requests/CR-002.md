# CR-002 — deterministic-machine: lift deterministic work out of the stochastic layer

> Change request against self-host aPRD v4 (`aprd.v4.frozen.md`). Bumps to v5. Re-triggers Phases 0·2·3·1 — the **sanctioned immutability exception** (workflow §4: change = new version + change request, never hand-edit frozen body), because it amends frozen canon (AB3/AB5/PR2 + prompt-skeleton) + adds frozen artifact classes. Design corpus authored + frozen at `_deterministic-machine/` (docs 00–04) — lifted, NOT re-derived (P5/B11). Caveman register.

## Ask
System today = **100% stochastic**: 39 prompts, every step an LLM clean-room session. Findings (doc 00):
- **39/39 steps bind a closed schema.** LLM only fills values; schema authoring + validation = pure code today spent on tokens.
- **Deterministic decisions embedded everywhere** — verdict-from-count, route-from-axes, topo-sort, monotonic-id, set-membership, threshold, bijection. All computable; all currently LLM tokens.
- Tiering: **~10 steps ~whole-mechanical (Tier 1)** · **~12 carry a liftable deterministic substage (Tier 2)** · **~17 stay LLM but gain a code validation wrapper (Tier 3)**.

Replace the stochastic mechanism with deterministic CODE **where the work is deterministic**; leave the LLM only the judgment (the Tier-3 islands).

## Scope delta (what v5 adds over v4)
Deterministic spine — three layers (doc 04): **frozen contracts** (schema-registry + io-manifest + playbooks) → **deterministic spine** (resolve · prefill · emit · compute · validate · lint, no LLM) → **stochastic islands** (LLM fills judgment holes only). New tracked builds = the **W0–W8 wave** (doc 04): canon CR → schema-registry+validate → io-manifest+resolve → graph-lint → det/ modules → Tier-1 emitters → re-author 39 prompts → orchestrator swap → pack gate. Reuses the one spine + overlay + playbook + both-directions-oracle pattern (P3) — no new pipeline architecture beyond the spine itself.

New `R*` ids (above high-water R8, BF3): **R9–R14** (§"NEW REQUIREMENTS" in v5).
