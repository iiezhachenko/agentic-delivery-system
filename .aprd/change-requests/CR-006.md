# CR-006 — orchestrator on-accept scratch cleanup: promote does not clean working files

> Operator-reported defect (2026-06-12). Class: bugfix. Change against `prompts/_orchestrator.md` (loop body only; no frozen artifact, no schema, no lock change required).

## Defect

STEP 6 (promote) atomically moves the accepted scratch `.md` to `prompts/<NN-phase>/<ROLE>.md` but does NOT remove the remaining scratch working files:
- `lint.json` (Layer 1 output)
- `economy-audit.json` (Layer 2 output)
- runner/verifier input `.txt` files
- `both-directions.json` / both-directions artefacts
- any superseded re-author `.md` scratch variants

These accumulate across builds. Evidence: session W12/W15 — scratch accumulated and required a SEPARATE explicit operator maintenance dispatch to remove. Desired: accept ⇒ promote artifact AND clean ALL scratch working files for that build, atomically, in the same accept path.

## Root cause

STEP 6 reads: "atomically move the scratch `.md` … That promotion IS the ship." No cleanup instruction follows. The step ends at the move; the working files are never mentioned.

The RULES block reinforces "no bookkeeping file, ever" (aimed at status/changelog ceremony), but this is not bookkeeping ceremony — it is failing to remove build temporaries after they are no longer needed. The distinction is absent, so STEP 6 silently omits cleanup.

## Fix scope

One targeted edit — one home (AB1):

**STEP 6** gains a cleanup clause immediately after the atomic move succeeds: remove all scratch working files for this build (`lint.json`, `economy-audit.json`, runner/verifier input `.txt` files, both-directions artefacts, superseded scratch `.md` variants). Cleanup fires ONLY on accept/promote (the move succeeded). NOT on reject (scratch discarded, no promotion occurs). NOT before the move succeeds (order: move first, then clean).

No RULES mirror needed: cleanup is procedural (a step in the defined path), not an unconditional prohibition needing a RULES guard. CR-005 pattern applies when the gap is a missing prohibition that could be violated anywhere; here the gap is a missing cleanup step in a single path — one home is correct.

## Out of scope

- Reject path — scratch is discarded before any promotion; no cleanup mandate needed (it is already gone).
- Frozen artifacts, schemas, locks — unchanged.
- Generic orchestrator (`prompts/_orchestrator.generic.md`) — separate loop body; audit deferred.
- `.roadmap/` bookkeeping — no new build id needed (this is a direct spine edit, P3 precedent W7/W9/W8/W15).

## Class

bugfix. Root cause = missing step in defined accept/promote path. Minimal fix = add the cleanup instruction; no logic change, no new capability.

## Status

Open (2026-06-12). Frontier = this CR; fix targets `prompts/_orchestrator.md` STEP 6.
