# CR-010 — reranker ledger hygiene: prune merged-wave records to keep the ledger relevant

> Operator-directed (2026-06-12), gate Q1–Q3 answered before dispatch. Change against ADP operating model (orchestrator STEP 0 + spine helper tools). Class: feature-add. Caveman register.

## Ask

Reranker file (`.roadmap/08-rerank.json`) accumulates forever: `completed[]` keeps every shipped wave (W0–W23), `coverage.completed[]` mirrors it, and `_note` grew to ~20 KB of `roadmap_version` 6→31 history. Stale records bury the live state. Operator wants: when a workstream/initiative is **complete, accepted, AND merged** (to master), its records are removed from the reranker so the file reflects only live + in-flight state.

## Gate answers (D20 guarantee 6 — persisted, never re-asked)

- **Q1 Scope:** FULL CR cascade — aPRD change-request (R-ids) → ADR (D31 ledger-hygiene decision) → HLD (component + contract delta) → Build (spine prune tool + orchestrator wire + oracle).
- **Q2 Trigger:** AUTO at STEP 0.1 reconcile. "Merged" = a `completed[]` entry's `done_sentinel` is present on `master` (`git cat-file -e master:<sentinel>`). Prune fires right after `git merge origin/main --ff-only` clean.
- **Q3 Retention:** COLLAPSE, git is the trail. Remove merged `completed[]`/`coverage.completed[]` rows + reset `_note` to a one-line current-state pointer (drop the 6→31 history). PRESERVE open `deferred_findings` + `remaining_sequence`/`remaining_ranked` + live `in_progress`. Audit trail = git history + shipped disk (D20).

## Scope delta (what this CR adds)

### A — Ledger-hygiene requirement (operating-model requirement)

The reranker ledger is **informational ordering, not the source of truth** (D20 — disk is truth; loop derives frontier from disk sentinels, never from this file). So merged records are pure noise: pruning them loses nothing the loop relies on. Records become prunable exactly when their wave is **complete + accepted + merged** — proven by the wave's `done_sentinel` existing on `master`.

**New R-ids (appended to aPRD requirements via this CR doc, same as CR-007's R-MW-*; monolithic `aprd.frozen.md` not re-frozen):**
- `R-LH-1` — **Prune-on-merge:** a `completed[]` entry whose `done_sentinel` is present on `master` is removed from the ledger (`completed[]` + `coverage.completed[]` mirror). Merged = complete + accepted + landed on the stable branch.
- `R-LH-2` — **Preserve live:** open carry-forward records are NEVER pruned — `remaining_sequence`, `remaining_ranked`, open `coverage.deferred_findings`, `coverage.missing`/`duplicated`, live `in_progress` survive every prune. Pruning is merged-only.
- `R-LH-3` — **Auto + idempotent at reconcile:** prune fires at STEP 0.1 after auto-reconcile from main; re-running on an already-pruned ledger is a no-op (last-writer-wins, D20 guarantee 4). Prune never alters the disk-derived frontier (D20) — it only collapses the informational ledger.

### B — Spine prune tool (`tools/det/prune-ledger.mjs`)

Deterministic helper (D22 sanctioned class). Pure core `pruneLedger(ledger, mergedSentinels)` → pruned ledger; CLI gathers `mergedSentinels` via `git cat-file -e master:<sentinel>` per `completed[]` row. Collapses `_note` to a generated one-line pointer; bumps `roadmap_version`. Both-directions selftest `prune-ledger.selftest.mjs` reads `_fixtures/ledger-prune/` golden. Auto-joins the W8 pack gate's `tools/**/*.selftest.mjs` sweep.

### C — Orchestrator delta (`prompts/_orchestrator.md`)

STEP 0.1 gains a prune step after `git merge origin/main --ff-only` clean: run `node tools/det/prune-ledger.mjs .roadmap/08-rerank.json --root .`. Mechanical git + JSON surgery — controller-direct (P3, RM11 carve-out: the orchestrator IS the control loop, not a stochastic-island deliverable; W7/W8/W18 precedent).

### D — HLD component/contract delta

New component **LEDGER-PRUNE** in `components.json` `inline_orchestrator[]` (operating-model helper, det-tool-backed, no prompt file — same pattern as STREAM-MANAGER). New STEP-0.1 postcondition contract in `contracts.json` (generated-frozen → edit `graph-lint.mjs::generateContracts` constant, regen, pass deep-equal selftest).

### E — ADR (ledger-hygiene decision)

New ADR-0031 (D31): why prune merged records (ledger = informational, D20), when (sentinel-on-master = merged), what survives (open/live), retention (collapse, git is the trail), tradeoffs (lose hot-file history vs relevance; git/disk retain truth).

## Out of scope

- Merging workstream branches to master (operator action — unchanged; prune only READS master to detect merged state).
- Preserving `_note` drift-class lessons elsewhere — those are reusable engineering knowledge but belong in canon (`.hld/`), not the rerank ledger; relocating them is a separate canon-home decision. Git history retains the old `_note`.
- Archiving pruned rows to a sidecar file (operator chose collapse-with-git-trail over archive-then-collapse).
- Generic orchestrator (`prompts/_orchestrator.generic.md`) — its product-slice ledger lifecycle is operator/demo-driven; ledger-prune applicability deferred (separate CR if needed).
- The strict product-slice `08-rerank.schema.json` — the self-host ledger is a deliberate repurposing (entries = prompt-builds), informational not schema-validated; prune preserves its self-host shape.
