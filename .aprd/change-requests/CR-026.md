# CR-026 — Role .md thinning: strip server-owned prose from role files

> Standalone. Operator-directed (2026-06-13). Workstream: `feature/mcp-adp-powers`. Source: `_self-host/00-analysis.md` §6 (P4). Caveman register.

## Ask

After CR-023 (driver loop) + CR-024 (projector), the MCP server owns: mode-dispatch, guard/`escapes:` eval, IO resolution + projection, schema-shaping (prefill). Role `.md` files currently carry all of this as prose the LLM reads verbatim. That prose is redundant duplication — the server resolved those decisions before the model call; the model carrying them re-reads dead weight.

Role `.md` post-thinning: **judgment prose only** — the text the model needs to fill its holes. Everything the server already handles → deleted. Token win: role verbatim ~3k → judgment-only ~1k per step. Applies to all 30+ shipped roles.

Gate: each thinned role MUST pass the SAME `_fixtures/` oracle at the same bar (both-directions: known-good PASS + planted-defect FAIL). Thinning that breaks the oracle = incorrect thinning — revert + re-scope.

## Scope delta

### A — Strip from each role .md (server-owned sections)

For each of the 30+ roles in `prompts/*/`:
1. **Mode-dispatch block** — (`SKELETON-BUILD vs SLICE-BUILD`, `class=greenfield vs bugfix`, branch logic) → server resolves before call; delete.
2. **`escapes:` / guard section** — (`lock missing OR status!=frozen OR …`) → `adp_guard` evaluates server-side; delete.
3. **Inline schema tables** — artifact shape prose the model inferred from → `adp_next` ships prefilled shell; delete.
4. **I/O frontmatter** — `outputs[].path`, `outputs[].schema` id declarations → server-known from manifest; delete.
5. **Spine tool call instructions** — ("run `node tools/det/verdict.mjs`…") → server calls those tools; delete.

**Keep:** everything the model's JUDGMENT needs. Finding prose. AC text templates. Scoring rubric. Adversarial directives. Narration instructions. Role mandate.

### B — Per-role verify cycle

Each thinned role `.md`: run through STEP 4 full verify (lint → ECONOMY-AUDIT → clean-room sim) against `_fixtures/`. Both-directions must hold at same bar as original shipping. Verify gate = same oracle; do not lower it for the refactor.

Batch order: Phase 0 (aPRD roles) → Phase 1 (Roadmap) → Phase 2 (ADR) → Phase 3 (HLD) → Phase 4 (Build). Within phase: CRITIQUE/adversarial roles last (most judgment-heavy, smallest strip benefit, highest verify sensitivity).

### C — `prompts/` skeleton reference update

`.hld/skeleton/prompt-skeleton.md` — if it documents the sections that are being stripped, update to reflect new thinned structure. Emit note: sections A–E above are now server-side; only judgment prose + holes template live in role `.md`. Frozen skeleton update = new version if applicable (CLAUDE.md: frozen artifact → new version + CR, already this CR).

### D — `_step-runner.md` / `.claude/agents/step-runner.md` update

Runner no longer receives mode-dispatch / guard / IO prose in the role `.md`. Runner receives: judgment-only prose + prefilled shell + projected input slices + holes[] from `adp_next`. Update runner contract doc to reflect thinned packet shape. Behavior unchanged; context smaller.

## Affected downstream
