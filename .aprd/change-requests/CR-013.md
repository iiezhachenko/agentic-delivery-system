# CR-013 — Three-tier write surfaces: controller zero-write, orchestrator mechanical-write, step-runner authoring (corrects ADR-0033 conflation)

> Operator-directed (2026-06-12), gate ACCEPT received before dispatch. Correction of CR-012/ADR-0033 (kills controller/orchestrator conflation; establishes three distinct entities with distinct write surfaces). Class: correction. Caveman register.

## Ask

CR-012/ADR-0033 introduced a structural conflation: D33 decision item 1 wrote "Controller (orchestrator)" — a parenthetical that equates two distinct entities. Mechanical writes (git ops, spine shell-outs, lock/index re-sign, scratch→promotion) were placed under R-CW-1 scoped to "Controller" — wrong. Controller = the Claude Code session from which `/evolve` is invoked. It writes NOTHING. The mechanical writes belong to the orchestrator (adp-orchestrator agent), not the controller. Operator directed: establish three-tier write surfaces with zero overlap; supersede ADR-0033 with ADR-0034; preserve all three R-ids (rewrite meanings).
