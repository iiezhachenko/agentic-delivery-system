# CR-023 — MCP driver loop: bake STEP 0–6 control logic into code

> Standalone. Operator-directed (2026-06-13). Workstream: `feature/mcp-adp-powers`. Source: `_self-host/00-analysis.md` §3–§6. Caveman register.

## Ask

`_orchestrator.md` STEP 0–6 = 142-line prose LLM executes every run. Every session loads: orchestrator (~15k tokens) + io-manifest (740 ln) + schemas (48) + mode-dispatch + branch-enforce + gate-sequencing. That prose IS the token floor — no technique removes it while the LLM drives.

Fix = invert control. "ADP Powers" MCP server owns the loop in code. Model gets called ONLY to fill judgment holes — nothing else in context. Existing spine modules (`tools/det/*`, `tools/io/resolve.mjs`, `emit/*`) become MCP tool bodies UNCHANGED (P3 engine-invariant). Net-new = driver loop wrapper + tool surface exposing existing modules.

Leaks closed (from `00-analysis.md` §2.1):

| # | Leak | Token weight | Moved to |
|---|---|---|---|
| L1 | control loop STEP 0–6 | high | driver code |
| L2 | frontier derivation: scan sentinels, schema-validate, class-discriminator | high | `adp_status` / `adp_next` |
| L3 | branch enforce A–E, auto-reconcile, ledger-prune | med | `adp_branch` |
| L9 | gate-verdict sequencing Layer1→Layer2→Layer3, short-circuit | med | `adp_submit` |

## Scope delta

### A — Driver loop (`adp-server/driver.js` or equivalent)

New code-layer driver implementing STEP 0–6 logic. Reads disk (frontier scan), calls `adp_next` to assemble the step packet, dispatches model as hole-filler function, calls `adp_submit` on reply, calls `adp_promote` on gate-accept. Idempotent: disk-derived, resume-safe (D20). No tracker writes.

### B — MCP tool surface (wrap existing spine modules)

New tools exposed by the MCP server — all backed by EXISTING code, no logic change:

| Tool | Wraps | Returns |
|---|---|---|
| `adp_status` | STEP 0 sentinel scan + frontier derive | frontier tally + next role |
| `adp_next` | STEP 0–2 + resolve + prefill | step packet (role, state, judgment_prose, inputs[], shell, holes[], output_path, schemaId) |
| `adp_emit` | `tools/det/emit/*` | whole Tier-1 artifact, validated |
| `adp_submit` | prefill splice + validate + lint + value-check | gate verdict + route |
| `adp_promote` | STEP 6 atomic move + cleanup | new frontier |
| `adp_branch` | STEP 0.0/0.1/0.1b | branch action / HALT |
| `adp_verdict` / `_route` / `_sequence` / `_idgen` / `_coverage` | `tools/det/*` as-is | decision only |
| `adp_guard` | `escapes:` predicate eval over disk | tripped? + which |

### C — Existing selftests remain

Spine modules unchanged → their selftests unchanged. `graph-lint.selftest.mjs`, `resolve.selftest.mjs`, all `tools/det/*.selftest.mjs` continue to pass. Server is a thin wrapper, not a reimplementation.
