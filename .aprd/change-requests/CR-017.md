# CR-017 — AUDIT-REPORT split: extract AUDIT-PROMOTE role (intake write)

> Parent: CR-016. Operator-directed (2026-06-12). Deferred — execute in subsequent build workstream after ADR-0037 acceptance. Caveman register.

## Ask

AUDIT-REPORT writes 2 unrelated outputs: `.audit/report.md` (operator-facing audit report) and `.aprd/00-raw-request.md` (Phase-0 ADP intake artifact). Different downstream consumers, different schemas, different phases. Separable functions: produce audit report vs generate ADP intake artifact. SRP block F1 (LC1) + F6 (LC3).

Split: AUDIT-REPORT keeps report only (`.audit/report.md`). New AUDIT-PROMOTE role writes `.aprd/00-raw-request.md` (promote path only).

## Findings

| id | criterion | severity |
|---|---|---|
| F1 | LC1 — 2+ unrelated output paths | block |
| F6 | LC3 — 2+ separable responsibilities, no single primary | block |

## Scope delta

### A — Modify AUDIT-REPORT

`prompts/00-aprd/AUDIT-REPORT.md` — remove promote path (Part B / interactive promote step that writes `.aprd/00-raw-request.md`). Primary output: `.audit/report.md` only. Stop condition: report written, no promote gate.

### B — New role: AUDIT-PROMOTE

`prompts/00-aprd/AUDIT-PROMOTE.md` — new role. Reads `.audit/audit-report.json` + `.audit/lenses.json`. Writes `.aprd/00-raw-request.md` shaped as ADP intake. Block findings → bugfix-class requirements; warn → feature-add-class requirements. Interactive: operator reviews, no auto-execution. Invoked optionally after AUDIT-REPORT, not inline.

### C — Playbook update

`prompts/_playbooks/audit-spine.md` — update promote_path entry: chain AUDIT-PROMOTE as optional post-report step (not embedded in AUDIT-REPORT).

### D — Spine updates

`tools/io/io-manifest.json` — add AUDIT-PROMOTE role entry (inputs: audit-report.json + lenses.json; outputs: .aprd/00-raw-request.md). `tools/io/components.json` — add AUDIT-PROMOTE component entry.

## Affected downstream

- AUDIT-REPORT callers: no output-path change for `.audit/report.md`; promote path removed.
- Any automation that expected AUDIT-REPORT to also write intake → must now chain AUDIT-PROMOTE.
- `audit-spine.md` playbook references AUDIT-REPORT for report + promote; update to reference AUDIT-PROMOTE for promote.

## Class

refactor. Modifies existing AUDIT-REPORT + adds AUDIT-PROMOTE. No schema changes to report output.
