## Register (governs ALL prose everywhere ‚Äî narration AND artifacts AND code comments)

Think, write, reply terse like smart caveman. All technical substance stays. Only fluff dies. Caveman is mandate, not chat-only register ‚Äî saves tokens at scale across whole tree.
- Drop: articles (a/an/the), filler (just/really/basically), pleasantries, hedging.
- Pattern: [thing] [action] [reason]. [next step]
- NOT: "Sure! I'd be happy to help you with that."
- YES: "Bug in auth middleware. Fix:"
- **Applies ABSOLUTE ‚Äî NO exception:** chat/narration, ALL artifact prose (spec/ADR/HLD/prompt/doc bodies), code comments ‚Äî incl human-facing artifacts. (Condensed reads faster; every artifact is also agent-ingested context.) Need different prose for a human consumer ‚Üí a separate agent OUTSIDE the pipeline restyles that one artifact; NEVER relax caveman inside the system.
- **Stays literal (not caveman, never corrupted):** structural data (JSON/YAML keys+values, schemas), identifiers (`R*`/`AC*`/`C*`/`ADR-*`/ids), code syntax itself. Caveman is register for prose; shortens comments, never breaks data or code.
- **Register + Economy = two SEPARATE absolutes, both consumer-independent.** Register = terse style (this section). Economy = substance discipline (one home per fact ¬∑ every statement earns its place ¬∑ single interpretation). Independent: terse-but-bloated still fails (economy), economical-but-full-prose still fails (register). Both bind every artifact.

## Standing conventions

- **One role = one prompt.** Role separation load-bearing (failure isolation). Split role further only when justified.
- **Artifacts land on disk.** Every prompt instructs agent to WRITE output to spec-defined path. Deliverable = file on disk, not chat reply.
- **Disk is source of truth.** All state derived from on-disk trees + `prompts/` + locks ‚Äî never from hand-maintained tracker. Writes atomic; resume re-derives frontier.
- **IDs thread end-to-end:** `R ‚Üí AC ‚Üí S ‚Üí ADR ‚Üí C ‚Üí CT ‚Üí F ‚Üí commit`.
- **LLM reconciles/verifies, never authors truth.** Cheapest-source-first; model specializes canon to contract, canon never source of truth.
- **Adversarial roles stay hostile** (GAP-DETECT, CRITIQUE, anti-cheat).
- **Author against DRY skeleton** (`.hld/skeleton/prompt-skeleton.md`): one home per fact (AB1‚ÄìAB6 in `.hld/skeleton/coding-canon.md`). Substance invariant; only duplication dies.
- **No harness MEMORY ‚Äî ever.** Prohibited from the Claude-Code memory store. ALL decisions, notes, caveats land in ADP on-disk trees: decisions ‚Üí `.adr/`, build/frontier state + fix records ‚Üí `.roadmap/`, class rules ‚Üí `_playbooks/`, requirements ‚Üí `.aprd/`, design ‚Üí `.hld/`. Disk is the sole source of truth (D20); a note outside these trees does not exist.

## Immutability & verify

- **Never overwrite a frozen artifact.** `.aprd/aprd.frozen.md`, `.hld/skeleton.frozen.md`, `.adr/log/` bodies + every `*.lock` are signed + immutable. A change = new version + change request re-triggering affected downstream stages.
- **Generated-frozen artifact ‚áí amend its GENERATOR, never the frozen copy alone.** Some frozen artifacts are emitted by a spine tool, not hand-written ‚Äî they carry a deep-equal verify that breaks silently if the frozen file drifts from its generator. Canonical case: `.hld/skeleton/contracts.json` is generated by `tools/io/graph-lint.mjs::generateContracts` (embedded constants for non-derivable prose); `graph-lint.selftest.mjs` asserts generated == frozen. Hand-editing the frozen copy without updating the generator drifts that gate (CR-008 did exactly this: added `preconditions` to the frozen file, left the generator behind). Fix forward: edit the generator/its constants, regen (`graph-lint.mjs --write`), confirm the selftest's deep-equal passes. Same shape for any future generated-frozen artifact.
- **Verify before done.** Prompt judged by running it clean-room against `_fixtures/` (oracle): known-good prompt PASSes, planted-defect copy FAILs. Both directions must hold. Schema-valid + ID-threaded + acceptance satisfied = bar.

## Verification authority ‚Äî operator runs every demo (IRON LAW, D39)

- **Agent NEVER runs the demo/proof it presents ‚Äî the verifying human runs it.** Any demonstration or acceptance verification of working behavior = operator-executed, ALWAYS. At every gate the agent's job: hand the operator EXPLICIT, copy-pasteable reproducible steps (exact commands + exact expected output), then STOP. Operator executes, observes, confirms. Agent-run output is NEVER the acceptance proof ‚Äî an agent running its own proof = self-grading (extends "LLM verifies, never authors truth" + the operator gate + adversarial separation). The human is the sole verifying oracle for demos.
- **Scope:** binds every launcher (`/evolve`, `/deliver`) + every gate + every "show it works" moment. NO exception, no "but I already ran it", no "scripting shortcut" that substitutes agent execution for the operator's. Internal clean-room build-loop selftests still run during authoring ‚Äî they are NOT the demo; the demo the human signs off on is operator-reproduced.
- **Acceptance demo = FRESH Claude Code session, NATIVE MCP integration.** The operator's demo MUST run from a fresh Claude Code session that loads the registered `.mcp.json` server and calls the `mcp__<server>__*` tools natively. That is the ONLY thing that proves the full wiring end-to-end ‚Äî `.mcp.json` correct + host connects + tools exposed + callable. Raw-protocol piping (`printf ‚Ä¶ | node server.mjs`) or a direct server spawn is a BUILD-TIME check only; it bypasses the host wiring and is NOT an acceptance demo. Same-session reconnect (`/mcp`) is the minimum; a brand-new session is the gold standard.
- **Failure mode this closes:** agent narrates "tests pass / it works" from its own run (or a direct-spawn shortcut) ‚Üí human trusts an unverified, cherry-picked, or unwired result. Fix: agent supplies the runnable steps; the human runs them from a fresh session through the real integration; nothing is "verified" until the human ran it that way.
